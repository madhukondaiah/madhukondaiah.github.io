⚠️
Common Mistakes to Avoid
  • Confusing List (mutable) with Tuple (immutable) in Python
  • Mixing up chmod (permissions) vs chown (ownership)
  • Thinking POST=update — POST=create new, PUT=update existing
  • Confusing precision and recall in ML metrics
  • Confusing SSR (server renders HTML) with CSR (browser renders via JS)
  • Thinking BigQuery is a regular database — it's a serverless data warehouse
  • HTTP 401 = Unauthenticated, 403 = Unauthorized/Forbidden (very commonly confused)

Python Programming
Data Structures
List — mutable, ordered, allows duplicates: [1,2,3]
Tuple — immutable, ordered: (1,2,3)
Dict — key-value, mutable: {"a":1,"b":2}
Set — unordered, no duplicates: {1,2,3}
Common List Methods
lst.append(x) # add to end lst.insert(i,x) # insert at index lst.pop() # remove last lst.remove(x) # remove first x lst.sort() # sort in-place lst[1:3] # slicing
Dict Methods
d.keys() # all keys d.values() # all values d.items() # key-value pairs d.get('k','default') # safe get
File I/O
with open('file.txt', 'r') as f: data = f.read() # Modes: r=read, w=write, a=append # rb=read binary, wb=write binary

Linux Commands
Permissions (chmod)
Read=4, Write=2, Execute=1 (add for permissions)
chmod 755 file # rwxr-xr-x chmod 644 file # rw-r--r-- chmod 777 file # rwxrwxrwx (all access) chmod +x file # add execute for all chmod u+w file # add write for owner # u=user/owner, g=group, o=others, a=all
Ownership (chown)
chown alice file.txt # change owner chown alice:devs file.txt # owner + group chown -R alice /dir # recursive chgrp devs file.txt # change group only
Networking Commands
netstat -tuln # show listening ports netstat -an # all connections ip route show # display routing table ip addr show # show IP addresses ifconfig # network interface config ping google.com # test connectivity traceroute host # trace packet path
File System Commands
ls -la # list all files with details ls -lh # human-readable file sizes ls -lt # sort by modification time find / -name "*.txt" # search files du -sh /dir # disk usage df -h # disk free space
Kernel Role
The kernel is the core of the OS. It manages: CPU scheduling (processes), memory management, device drivers, file system, system calls (interface between apps and hardware). The kernel runs in privileged mode; user apps run in user mode.
Process Management
ps aux # list all processes kill -9 PID # force kill process top / htop # live process monitor systemctl start nginx # start service systemctl status nginx # check service

Database & SQL
ACID Properties
Atomicity — Transaction is all-or-nothing. If one part fails, entire transaction rolls back.
Consistency — Database always moves from one valid state to another.
Isolation — Concurrent transactions don't interfere with each other.
Durability — Committed data persists permanently, even after system crash.
SQL CRUD Commands
-- Create table CREATE TABLE users (id INT, name VARCHAR(50)); -- Insert INSERT INTO users VALUES (1, 'Alice'); -- Read SELECT * FROM users WHERE age > 25; SELECT name, age FROM users ORDER BY age DESC; -- Update UPDATE users SET age=30 WHERE id=1; -- Delete DELETE FROM users WHERE id=1;
JOINs
-- INNER JOIN: matching rows in both tables SELECT u.name, o.amount FROM users u INNER JOIN orders o ON u.id = o.user_id; -- LEFT JOIN: all rows from left table -- GROUP BY with aggregation SELECT dept, COUNT(*) as total FROM employees GROUP BY dept HAVING COUNT(*) > 5;
Cypher (Neo4j Graph DB)
Cypher is the query language for Neo4j, a graph database where data is stored as nodes and relationships.
-- Find a node MATCH (n:Person {name:'Alice'}) RETURN n; -- Find relationships MATCH (a:Person)-[:KNOWS]->(b:Person) RETURN a.name, b.name; -- Create node CREATE (n:Person {name:'Bob', age:30});
SQL vs NoSQL
SQL (Relational): Structured tables, fixed schema, ACID compliant. E.g., MySQL, PostgreSQL.
NoSQL: Flexible schema, scales horizontally. E.g., MongoDB (document), Neo4j (graph), Redis (key-value), Cassandra (column).

REST API & Web Technologies
HTTP Methods
GET — Retrieve/read data (no body)
POST — Create a new resource (data in body)
PUT — Update/replace entire resource
PATCH — Partial update of a resource
DELETE — Remove a resource
HTTP Status Codes
200 OK — Request succeeded
201 Created — Resource created (POST)
204 No Content — Success, no body returned
400 Bad Request — Invalid input from client
401 Unauthorized — Not authenticated (no/invalid token)
403 Forbidden — Authenticated but no permission
404 Not Found — Resource doesn't exist
500 Internal Server Error — Server crashed
JSON Format
// JSON = JavaScript Object Notation // Key-value format, human-readable { "id": 1, "name": "Alice", "roles": ["admin", "user"], "address": { "city": "Chennai", "country": "India" } }
SSR vs CSR
SSR (Server-Side Rendering): HTML built on the server, sent to browser. Faster first load, better SEO. Used in: Next.js, PHP, Django templates.

CSR (Client-Side Rendering): Browser downloads JS bundle, then builds HTML. Initial load slower but very interactive. Used in: React, Angular, Vue SPAs.
API Gateway & Fuzz Testing
API Gateway: Single entry point for all client requests. Handles routing, authentication, rate limiting, load balancing. Examples: AWS API Gateway, Kong, NGINX.

Fuzz Testing (Fuzzing): Automated security technique that sends random/unexpected/invalid inputs to an application to find crashes, memory leaks, or vulnerabilities.
HTTP Auth Methods
Basic Auth: Base64 encoded username:password in header.
Bearer Token / JWT: Token in Authorization header after login.
OAuth 2.0: Delegated access (login with Google/Facebook).
API Key: Secret key passed in header or query param.
CSS Flexbox Essentials
.container { display: flex; flex-direction: row; /* or column */ justify-content: center; /* main axis */ align-items: center; /* cross axis */ flex-wrap: wrap; gap: 16px; }

AI / ML Concepts
ML Model Lifecycle
1. Data Collection — Gather raw data from sources
2. Preprocessing — Clean, normalize, encode, handle missing values, feature engineering
3. Model Training — Feed data to algorithm, model learns patterns
4. Evaluation — Test on unseen data, measure accuracy/loss
5. Deployment — Serve model via API for production use
6. Monitoring — Track performance drift, retrain when needed
Confusion Matrix
TP (True Positive) — Predicted positive, actually positive ✅
TN (True Negative) — Predicted negative, actually negative ✅
FP (False Positive) — Predicted positive, actually negative ❌ (Type I Error / False Alarm)
FN (False Negative) — Predicted negative, actually positive ❌ (Type II Error / Missed)

Accuracy = (TP+TN) / Total
Precision = TP / (TP+FP) — "of all predicted positive, how many were correct?"
Recall = TP / (TP+FN) — "of all actual positives, how many did we find?"
CNN (Convolutional Neural Networks)
Specialized neural network for image processing. Uses convolutional layers to extract spatial features (edges, textures, shapes) through filters/kernels. Architecture: Input → Conv layers → Pooling → Flatten → Fully Connected → Output. Used in: image classification, object detection, facial recognition, medical imaging.
Clustering Algorithms
K-Means: Unsupervised. Divides data into K clusters by distance to centroid. Iteratively reassigns points and recalculates centroids until stable. Applications: customer segmentation, document clustering.

Hierarchical Clustering: Builds a tree of clusters (dendrogram). No need to specify K upfront.

DBSCAN: Density-based. Finds clusters of arbitrary shape. Handles outliers well.
Descriptive Analysis
Summarises and describes historical data. Answers "what happened?". Key metrics: Mean, Median, Mode, Standard Deviation, Variance, Min/Max, Quartiles. Tools: Pandas describe(), histograms, bar charts.
Supervised vs Unsupervised
Supervised: Labeled training data. Goal: predict output. E.g., linear regression, decision trees, neural nets.
Unsupervised: No labels. Goal: find patterns/structure. E.g., K-means, PCA, autoencoders.
Reinforcement Learning: Agent learns by reward/penalty. E.g., game playing AI.

GCP (Google Cloud Platform)
BigQuery
Fully managed, serverless data warehouse. No infrastructure to manage. Runs SQL queries on petabyte-scale datasets. Columnar storage for fast analytics. Ideal for business intelligence, large-scale data analysis. Billed per query (bytes processed).
-- BigQuery SQL example SELECT country, COUNT(*) as users FROM `project.dataset.table` WHERE signup_date > '2024-01-01' GROUP BY country ORDER BY users DESC;
Load Balancer
Distributes incoming traffic across multiple backend instances. Types in GCP:
HTTP(S) LB — Global, Layer 7, routes based on URL/headers
TCP/UDP LB — Layer 4, regional
Internal LB — For traffic within VPC
Benefits: High availability, auto-scaling, health checks, SSL termination.
GCP Compute Commands
# List VM instances gcloud compute instances list # Create a VM gcloud compute instances create my-vm \ --zone=us-central1-a --machine-type=n1-standard-1 # SSH into VM gcloud compute ssh my-vm --zone=us-central1-a # Copy files (GCS) gsutil cp file.txt gs://my-bucket/ gsutil ls gs://my-bucket/
Key GCP Services
Compute Engine (VMs) Cloud Run (containers) Cloud Functions (serverless) GKE (Kubernetes) Cloud Storage BigQuery (analytics) Cloud SQL (managed DB) Pub/Sub (messaging)
Serverless Architecture
No server management. Cloud provider handles infrastructure. Auto-scales to zero. Pay per execution/use. Examples: BigQuery (serverless SQL), Cloud Functions (event-driven code), Cloud Run (containerized apps).

Networking
TCP/IP Stack
Application Layer — HTTP, HTTPS, FTP, DNS, SMTP
Transport Layer — TCP (reliable, ordered) | UDP (fast, no guarantee)
Network Layer — IP addressing, routing (routers)
Data Link — MAC addresses, switches, Ethernet
Physical — Cables, signals, hardware
TCP vs UDP
TCP: Connection-oriented, reliable, ordered, error-checked. 3-way handshake (SYN→SYN-ACK→ACK). Use for: HTTP, email, file transfer.
UDP: Connectionless, fast, no guarantee. Use for: video streaming, gaming, DNS, VoIP.
DNS (Domain Name System)
Translates domain names → IP addresses. Process: Browser → Recursive Resolver → Root Server → TLD Server → Authoritative Server → IP address returned. nslookup google.com to check DNS. Record types: A (IPv4), AAAA (IPv6), CNAME (alias), MX (mail), TXT (text).
IP Routing
Routers use routing tables to decide where to forward packets. Routes have: destination network, subnet mask, gateway, metric. Static routes = manually configured. Dynamic routes = learned via protocols (OSPF, BGP, RIP).
ip route show # view routing table route -n # numeric routing table ip route add 10.0.0.0/24 via 192.168.1.1
TCP/IP Troubleshooting Steps
1. ping 8.8.8.8 — check internet reachability
2. ping hostname — check DNS resolution
3. traceroute host — find where packets drop
4. netstat -tuln — check if port is open
5. Check firewall rules (iptables -L)
6. Check interface config (ip addr)
