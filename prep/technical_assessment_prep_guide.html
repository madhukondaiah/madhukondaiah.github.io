Strategy
How to approach the exam
1
Read each question fully before looking at options — MCQs often have trap options that seem right at first glance.
2
You have ~2 min per question. If stuck, eliminate 2 wrong options first, then make your best guess — don't leave blanks.
3
80% is the pass mark — that means you can afford to get 6–7 wrong out of 35. Don't panic if a few questions seem hard.
4
Watch for "NOT" or "EXCEPT" in questions — these flip the answer.
5
For Linux commands and SQL, focus on command syntax and what each flag/keyword does.
High-weight topics
Focus these first — appear most in assessments
1
Python basics — List, Dict, Tuples, file I/O, NumPy arrays. Know the difference between mutable and immutable types.
2
SQL fundamentals — CRUD (CREATE, SELECT, INSERT, UPDATE, DELETE), WHERE, JOIN, GROUP BY, ACID properties.
3
Linux commands — chmod, chown, ls, netstat, ip route. Know permission numbers (755, 644 etc.).
4
AI/ML pipeline — Data Collection → Preprocessing → Training → Deployment. Know what each stage does.
5
REST API concepts — HTTP methods (GET/POST/PUT/DELETE), status codes (200, 400, 404, 500), JSON format.
Quick Mnemonics
Memory tricks
A
ACID = Atomicity, Consistency, Isolation, Durability
B
chmod 755 = Owner: rwx, Group: r-x, Others: r-x. Think 4+2+1=7 (read+write+exec)
C
REST verbs: Get=Read, Post=Create, Put=Update, Delete=Delete (GPDD)
D
ML pipeline: "Don't Play The Damn Guitar" = Data → Preprocess → Train → Deploy

Take all practice questions in this guide. Revisit weak areas. Review all mnemonics. Sleep well the night before!
Python
Key facts to remember
• List = mutable, ordered. Tuple = immutable, ordered. Dict = key-value, mutable.
• len(), append(), pop(), get(), items(), keys() — common methods
• TensorFlow: used for ML model building; tf.constant, tf.Variable, model.fit()
• File I/O: open('file.txt', 'r') modes: r, w, a, rb
Linux
Critical commands
• chmod 755 file — sets rwxr-xr-x permissions
• chown user:group file — change file owner
• netstat -tuln — show listening ports
• ip route show — display routing table
• Kernel role: manages hardware, memory, processes, device drivers
Database
SQL + ACID + Cypher
• ACID: Atomicity (all or nothing), Consistency (valid state), Isolation (concurrent txns independent), Durability (persists after commit)
• SQL: SELECT * FROM table WHERE condition GROUP BY col
• Cypher (Neo4j): MATCH (n:Label) RETURN n — graph query
REST API / Web
HTTP + JSON + Rendering
• HTTP methods: GET (read), POST (create), PUT (update), DELETE (delete)
• Status codes: 200 OK, 201 Created, 400 Bad Request, 401 Unauthorized, 404 Not Found, 500 Server Error
• JSON: key-value text format, human-readable, {"key": "value"}
• SSR = HTML built on server. CSR = HTML built in browser via JS.
• Fuzz testing = sending random/invalid inputs to find vulnerabilities
AI / ML
Concepts and lifecycle
• Pipeline: Data Collection → Preprocessing → Training → Evaluation → Deployment
• Confusion Matrix: TP (correct positive), FP (false alarm), FN (missed), TN (correct negative)
• CNN: Convolutional layers extract image features; used in image classification
• K-Means clustering: groups data into K clusters by distance to centroid
• Descriptive analysis: summarizes historical data (mean, median, mode)
GCP
Google Cloud Platform
• Load Balancer: distributes traffic across multiple servers for availability
• BigQuery: serverless, fully managed data warehouse for analytics at scale
• Key commands: gcloud compute instances list, gsutil cp for storage
Networking
TCP/IP, DNS, IP routing
• TCP/IP: Transmission Control Protocol ensures reliable data delivery
• DNS: translates domain names (google.com) to IP addresses
• IP routing: routers forward packets based on routing tables
• Troubleshooting: ping (reachability), traceroute (path), netstat (connections)
